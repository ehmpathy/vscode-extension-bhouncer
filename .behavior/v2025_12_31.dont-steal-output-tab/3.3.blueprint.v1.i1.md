# blueprint: dont steal output tab

## summary

remove the automatic `outputChannel.show()` call during initialization. the output channel will still be created and populated with logs, but will not switch the bottom panel's active tab.

## root cause

in `src/domain.operations/output/createOutput.ts:48`:

```ts
config.outputChannel.show(true); // preserveFocus=true keeps editor focused
```

the comment is misleading — `preserveFocus=true` only prevents the *cursor* from moving to the output area. it does NOT prevent the bottom panel from *switching* its visible tab from terminal/problems/debug to output.

## change

### file: `src/domain.operations/output/createOutput.ts`

**before:**
```ts
  // initialize output channel if enabled
  if (input.enabled && input.createOutputChannel) {
    config.outputChannel = input.createOutputChannel('bhouncer');
    // show the output channel so users can see logs immediately
    config.outputChannel.show(true); // preserveFocus=true keeps editor focused
  }
```

**after:**
```ts
  // initialize output channel if enabled (channel exists but is not auto-shown)
  if (input.enabled && input.createOutputChannel) {
    config.outputChannel = input.createOutputChannel('bhouncer');
    // note: intentionally NOT calling .show() to avoid stealing panel focus
    // users can manually switch to output panel and select bhouncer from dropdown
  }
```

## why this works

| behavior | before | after |
|----------|--------|-------|
| output channel created | ✅ | ✅ |
| logs written to channel | ✅ | ✅ |
| channel available in dropdown | ✅ | ✅ |
| panel switches to output on init | ⛔ yes | ✅ no |
| user's terminal stays visible | ⛔ no | ✅ yes |
